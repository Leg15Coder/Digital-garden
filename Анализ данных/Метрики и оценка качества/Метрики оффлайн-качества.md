В [[Основные понятия МО|задачах бинарной классификации]] объекты разделяются на два класса: положительный (1) и отрицательный (0). Предсказания модели сравниваются с истинными ответами, на основе чего строится матрица ошибок:

![[Матрица_ошибок_в_машинном_обучении.png]]

Компоненты матрицы ошибок:
- **TP (True Positive)** - истинно положительные ответы.
- **TN (True Negative)** - истинно отрицательные ответы.
- **FP (False Positive)** - ложноположительные ответы (ошибка I рода).
- **FN (False Negative)** - ложноотрицательные ответы (ошибка II рода).

## Метрики качества классификации
### Accuracy

==Опр.== **Accuracy** - доля объектов, для которых модель верно угадала класс, среди общего числа объектов.$$\text{Accuracy} = \frac{TP + TN}{TP + TN + FP + FN}$$
- **Область значений:** $[0, 1]$
- **Ограничение:** Метрика абсолютно не репрезентативна в задачах с сильным дисбалансом классов. Например, если в выборке 99% объектов относятся к классу 0, то тривиальный константный классификатор, всегда предсказывающий класс 0, получит $Accuracy = 0.99$. При этом модель будет абсолютно бесполезна для поиска целевого класса 1.

### Precision (Точность)

==Опр.== **Precision (точность)** - доля объектов, действительно являющихся положительными, среди всех объектов, которые алгоритм отнес к положительному классу.$$\text{Precision} = \frac{TP}{TP + FP}$$
- **Смысл:** Показывает, насколько можно доверять модели, когда она утверждает, что объект принадлежит к классу 1. Метрика критична в задачах, где высока стоимость ложного срабатывания.

### Recall (Полнота)

==Опр.== **Recall (полнота)** - доля объектов положительного класса, которые алгоритм смог найти, среди всех истинно положительных объектов выборки.$$\text{Recall} = \frac{TP}{TP + FN}$$
- **Смысл:** Показывает способность алгоритма обнаруживать объекты целевого класса. Метрика критична в задачах, где нельзя допускать ложные пропуски событий.

### F1-мера

==Опр.== **F1-мера** - это среднее гармоническое между Precision и Recall, используемое для их агрегации в один сбалансированный критерий.$$F_1 = 2 \cdot \frac{\text{Precision} \cdot \text{Recall}}{\text{Precision} + \text{Recall}}$$
- **Свойства:** 
	* Среднее гармоническое обладает свойством жёстко штрафовать модель за проседание любой из двух метрик *(оно стремится к минимуму из двух величин)*.
    - Максимальное значение $F_1 = 1$ достигается только при идеальных $Precision = 1$ и $Recall = 1$

## ROC-AUC

==Опр.== **ROC-AUC (Receiver Operating Characteristic - Area Under Curve)** - площадь под ROC-кривой, которая комплексно оценивает качество ранжирования объектов моделью независимо от выбранного порога бинаризации.

### Геометрический смысл и построение

ROC-кривая строится в осях:
1. **TPR (True Positive Rate / Recall)** на оси Y: доля верно предсказанных положительных объектов среди всех истинно положительных. $$TPR = \frac{TP}{TP + FN}$$
2. **FPR (False Positive Rate)** на оси X: доля ошибочно предсказанных положительных объектов среди всех истинно отрицательных.$$FPR = \frac{FP}{TN + FP}$$
- Кривая получается путём плавного изменения порога бинаризации предсказанных моделью вероятностей от $1$ до $0$.
- Для случайного классификатора кривая совпадает с диагональю, а $ROC\text{-}AUC = 0.5$. Идеальный классификатор безошибочно разделяет классы, его кривая проходит через точку $(0, 1)$, а $ROC\text{-}AUC = 1.0$.

### Вероятностный смысл

**ROC-AUC** численно равен вероятности того, что случайно выбранный объект из положительного класса получит от модели оценку выше, чем случайно выбранный объект из отрицательного класса:$$ROC\text{-}AUC = P(f(x_0) < f(x_1)|y_0 = 0,\ y_1 = 1)$$

- Метрика инвариантна к балансу классов, поскольку формулы $TPR$ и $FPR$ нормируются независимо на полные объёмы своих классов ($TP+FN$ и $TN+FP$).

## Алгоритм вычисления ROC-AUC

Вычисление ROC-AUC на конечной выборке из $N$ объектов выполняется по следующему алгоритму:

1. **Сортировка:** Отсортировать все объекты выборки по убыванию предсказанных моделью вероятностей $f(x_i) = \hat{y}_i$.
2. **Подсчёт классов:** Найти общее количество истинно положительных объектов $TP$ и истинно отрицательных объектов $TN$ во всей выборке.
3. **Построение графика:**
    1) Начать движение из точки $(0, 0)$ на координатной плоскости $(FPR, TPR)$.
    2) Проходить по обратному вариационному ряду объектов ($\hat{y}_{(N)}, \dots, \hat{y}_{(1)}$). Пусть оригинальный номер (до вариационного ряда) текущего объекта был $k$, тогда:
        1) Если $y_k = 1$, сделать шаг вертикально вверх на величину $\frac{1}{TP}$
        2) Если $y_k = 1$, сделать шаг вертикально вверх на величину $\frac{1}{TN}$
	    3) Если у нескольких объектов предсказанные значения совпадают, шаг делается по диагонали в точку, соответствующую суммарному смещению за все совпадающие объекты.
4. **Интегрирование:** Рассчитать площадь получившейся фигуры.

### Аналитическая формула 
$$\text{ROC-AUC}\ =\ \frac{\sum\limits_{i :y_i = 1}\ \sum\limits_{j : y_j=0} I(f(x_i), f(x_j))}{TP \cdot TN}$$

где $I$ - обобщённый индикатор разности:$$I(a, b) = \begin{cases} 1, & a > b \\ 0.5, & a = b \\ 0, & a < b \end{cases}$$
