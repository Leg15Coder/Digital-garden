## Теорема о существовании [[Сложность алгоритмов по памяти||PSPACEC]]

- $SPACETMSAT = \{(M, x, s)\ |\ \text{МТ}\ M\ \text{принимает}\ x\ \text{используя не более}\ s\ \text{ячеек памяти}\} \in PSPACEC$

#### Доказательство

$NPCPASE = PSPACE$ $\Rightarrow$ $NPCPASEC = PSPACEC$. Опишем работу НМТ $M'$, распознающей $SPACETMSAT$:
- Входное слово поступает на специальную ленту, доступную только для чтения
- Имеется односторонняя лента, на которой будет запускаться эмулятор
- При получении входного слова, $M'$ ставит специальный символ на $s$-ю ячейку односторонней ленты для эмулятора. Если в какой то момент времени эмулятор попал на эту ячейку, то $M$ использовала хотя бы $s+1$ ячейку, поэтому $M'$ сразу же возвращает $q_{no}$
- Если за всё время эмуляции спецсимвол не был достигнут, то $M'$ возвращает $q_{yes}$

Память, которая тратится на запуск эмулятора, зависит только от кода МТ $M$, то есть линейна от входа. Сама эмуляция занимает не более $s$ ячеек, то есть линейна от входа $\Rightarrow$ $M'$ работает за линейную память, то есть $SPACETMSAT \in PSPACEC$

Пусть $A \in PSPACE$ и его разрешает ДМТ $M_A$, используя $S_A(n) = O(poly(n))$ ячеек памяти. Тогда пусть $f(x) = (M_A, x, S_A(|x|))$ - данная функция работает за $S_A(|x|) + |M_A| = S_A(|x|) + const = poly(|x|)$ $\Rightarrow$ $(x \in A \Leftrightarrow f(x) \in SPACETMSAT)$ $\Rightarrow$ $A \leq_p SPACETMSAT$ $\Rightarrow$ $SPACETMSAT \in PSPACEC$
