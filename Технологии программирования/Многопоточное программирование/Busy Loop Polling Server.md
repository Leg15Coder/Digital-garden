Ранее мы выяснили, что создание потока на каждого клиента приводит к нехватке ресурсов ОС (проблема C10k), и разработали [[Thread Pool]]. Однако, если мы используем базовый блокирующий ввод-вывод, потоки из пула быстро "повиснут" на системном вызове `read()`, ожидая данных от медленных клиентов. Пул истощится, и сервер остановится. Для решения этой проблемы сокеты переводятся в неблокирующий режим с помощью `fcntl`:
```cpp
int flags = fcntl(client_fd, F_GETFL, 0);
fcntl(client_fd, F_SETFL, flags | O_NONBLOCK);
```
Теперь поведение `read` и `write` меняется:
- Если данные есть - функция забирает их и возвращает количество байт.
- Если данных нет - функция не усыпляет поток, а **моментально возвращает `-1`**, при этом переменная `errno` устанавливается в значение `EAGAIN` или `EWOULDBLOCK` ("ресурс временно недоступен, попробуйте снова").

Сервер может использовать один поток, который хранит список всех подключенных клиентов (в `std::vector<int>`\) и в бесконечном цикле опрашивает их.

```cpp
// ... accept non-blocking сокетов и добавление в vector clients ...
while (server_is_running) {
    for (auto it = clients.begin(); it != clients.end(); ) {
        char buffer[1024];
        ssize_t bytes = read(*it, buffer, sizeof(buffer));

        if (bytes > 0) {
            // Данные получены, обрабатываем!
            work();
            ++it;
        } else if (bytes == -1 && (errno == EAGAIN || errno == EWOULDBLOCK)) {
            // Данных пока нет, идем дальше по списку
            ++it;
        } else {
            // Ошибка или EOF
            close(*it);
            it = clients.erase(it);
        }
    }
}
```

#### Проблемы такой реализации:
1. **Busy Waiting:** Поток ни на секунду не засыпает. Даже если сервер простаивает и никто ничего не пишет, ядро процессора будет загружено на 100%, выполняя бессмысленные `read`, возвращающие `EAGAIN`.
2. **Плохая масштабируемость ($O(N)$):** Если у нас подключено 10 000 клиентов, а данные пришли только от одного, потоку нужно сделать 10 000 дорогих системных вызовов, чтобы найти этого одного клиента.
